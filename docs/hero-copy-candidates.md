- CTA: `프로젝트 시작하기` / `포트폴리오 보기`

## ⚠️ Inversion (왜 실패하나 → 예방)

1. **A "출시까지"가 흔한 약속** → 히어로 직후 Story 섹션의 "미완성 프로토타입" 대비로 차별 입증. H1 단독으론 부족, 섹션 흐름으로 받쳐야.
2. **B "AI"가 막연** → 반드시 "검증된/풀사이클"로 한정. 막연한 AI 주장은 신뢰 하락.
3. **수치(3주 MVP 등) 미검증 시 역효과** → typewriter/sub에 구체 수치 넣으려면 testimonial 외 실제 사례 데이터 확보 후. 현 후보는 수치 미포함(안전).

## 시각 처리 (frontend-design)

- H1을 메시지로 바꾸면 브랜드명은 nav 로고에 이미 있어 중복 아님.
- 권장: eyebrow에 작은 `EVERGREEN DEV` 또는 `WHY EVERGREEN` + 큰 메시지 H1(기존 그라데이션 `.highlight` 스타일 일부 키워드에 적용) + slogan.
- typewriter는 유지(브랜드 시그니처) 하되 문구만 outcome 중심으로 교체.

## 적용 방법 (선택 확정 후)

1. `i18n/locales/ko.json` + `en.json` `hero` 블록 교체 (parity 유지)
2. typewriter는 `hero.typewriter.*` 키 4개 교체 (`pages/index.vue` 405행 computed 자동 반영)
3. H1 구조 변경 시 `pages/index.vue` 9-15행 + `.animate-text`/`.hero-slogan` CSS 조정
4. Playwright 데스크탑/모바일 시각 검증 → feat 브랜치 커밋
