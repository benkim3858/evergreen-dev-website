소비자 앱 · 매장 관리 앱 · 백오피스로 구성된 3면 O2O 플랫폼. 포장 주문 접수부터 PG 결제 연동, 자동 정산, 환불 처리까지 전 과정을 하나의 시스템으로 통합.

Period2025.10 – 2025.12

01
◎ Challenge

매장 · 소비자 · 본사가 따로 도는
분절된 F&B 포장 주문 생태계

기존 F&B 포장 주문 시장은 소비자 앱, 매장 POS, 본사 정산 시스템이 각기 다른 공급사 제품으로 분절되어 매장은 여러 도구를 동시에 관리해야 했고 결제-정산 과정에서 수기 대조가 필수였습니다. 이는 운영 비용 상승과 정산 오류 리스크로 직결됩니다.

핵심 과제

소비자·매장·본사 3면을 하나로 묶는 단일 O2O 플랫폼을 구축하고, 주문 접수부터 PG 결제·자동 정산·환불까지 전 과정을 하나의 시스템에서 처리하는 엔드투엔드 워크플로우 설계.

◎ Approach

3면 통합 아키텍처 설계

1
단일 Backend, 다중 Frontend

Nest.js 기반 단일 API 서버로 비즈니스 로직을 통합하고, Flutter로 소비자·매장 관리 앱을 구현해 개발 효율과 정합성을 동시 확보.

2
Flutter Web 백오피스

본사·운영자용 대시보드는 Flutter Web으로 구현해 코드 공유 극대화. 모바일·웹 컴포넌트를 한 저장소에서 관리.

3
PG 연동 + 자동 정산 로직

결제 승인·취소·환불 시나리오를 상태 머신으로 모델링하고, 매장별 자동 정산 배치 로직을 구현해 수기 작업 제거.

4
AWS 기반 확장 인프라

ECS/RDS/S3 기반 구성으로 매장 증가 시 수평 확장 가능. 실시간 주문 알림은 WebSocket + FCM 하이브리드로 안정화.
